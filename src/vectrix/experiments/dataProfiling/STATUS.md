# dataProfiling — 시계열 데이터 프로파일링 연구

## 정체성

이 실험 폴더는 **시계열 데이터의 본질을 이해하는 연구**다.

기존 실험(modelCreation, ensembleEvolution 등)은 "어떤 모델이 더 정확한가"를 물었다.
이 폴더는 그보다 앞선 질문을 던진다: **"이 데이터가 무엇인가?"**

## 핵심 원칙

1. **라벨 없이 시작한다** — "계절성 지배", "트렌드 지배" 같은 사전 분류를 만들지 않는다. 데이터가 스스로 그루핑을 보여줘야 한다.
2. **숫자만 본다** — DNA 특성 벡터를 정규화하고 내적(코사인 유사도)한다. 도메인 지식은 나중에.
3. **많이 본다** — 처음부터 분류 기준을 세우지 않는다. 충분히 관찰한 뒤에 패턴이 보이면 그때 라벨링한다.
4. **점진적으로 쌓는다** — 이 연구는 한 번에 끝나지 않는다. 실험을 거듭하며 이해를 깊게 한다.

## 장기 비전과의 연결

```
[이 폴더]                    →  중기                      →  장기
DNA 특성 벡터 그루핑            패턴 유형별 분류 정책          도메인 컨텍스트 통합
코사인 유사도 행렬              유형 → 최적 모델 매핑          "이게 뭔 데이터인지" 이해
차원 축소 시각화                자연어 인사이트 생성           LLM 기반 도메인 지식
클러스터 관찰                   profile() API                예측 인텔리전스 플랫폼
```

## 핵심 발견

### E001
- 빈도는 DNA 공간의 주요 축이지만, 같은 빈도 안에서도 0.37~0.49의 다양성
- PC1=자기상관(20.8%), PC2=스케일(18.7%), PC3=트렌드(10.5%), 95% 분산에 21개 성분
- Hourly 가장 동질적(0.494), Daily 가장 이질적(0.369)

### E002
- **원리 입증**: 계절성 지배 시리즈→CES, 트렌드 지배 시리즈→DOT (최적 모델이 다름)
- **실용성 미흡**: k=2 클러스터는 너무 조악, 실질 OWA 개선 ~0%
- **상한선**: per-series oracle OWA 0.665 → 현재 대비 15.6% 개선 가능 (이론적)
- **다음 과제**: 더 세밀한 분류, 다른 빈도(Yearly/Quarterly/Daily) 검증, 클러스터 해상도 향상

### E003
- **W·R 상관 확인**: seasonality_strength↔4Theta OWA(r=0.40), ↔ETS OWA(r=0.53) — 유의미
- **DOT/CES는 계절성에 둔감**(r=0.14~0.20) → 범용성 높은 이유를 DNA 수준에서 확인
- **Best model 분포**: DOT 39%, CES 26%, 4Theta 20%, ETS 14% — DOT가 다수지만 만능은 아님
- **Ridge 선형 매핑 실패**: 36.7% 정확도, DOT보다 악화(-30.9% oracle gap)
- **핵심 교훈**: 상관은 있지만 매핑은 비선형. 모델 간 성능 차이가 작아서 "잘못 고르면 악화"
- **agipath 비유**: W·R 내적 원리는 맞지만, 시리즈 수/맥락 풍부함이 NLP에 비해 부족

### E004
- **R 풍부화(horizon별) 효과 없음**: Ridge 정확도 39%→38%, OWA 0.848→0.845. 문제는 R 차원이 아님
- **단순 규칙 > Ridge**: seas>0.60→CES(0.807), acf_lag2<=0.37→CES(0.801) 모두 Ridge(0.845)보다 우월
- **다중 특성 규칙**: seas>0.5 & trend<=0.1→CES = OWA 0.803 (DOT 대비 -0.009)
- **Oracle gap 9%만 캡처**: 0.127 gap 중 0.011 회수 → 모델 간 OWA 차이가 작아서 선택 가치 제한적
- **잔차 인사이트**: CES-best 시리즈는 DOT 잔차에 자기상관 높음(0.61 vs 0.52)
- **근본적 결론**: Monthly 4모델에서 프로파일링 기반 모델 선택은 이론적으로 12.7% 개선 가능하나, DNA→모델 매핑의 정확도가 너무 낮아 실용적 가치는 ~1% (0.812→0.801)

### E005
- **Yearly oracle gap 35.6%로 최대** — DOT의 Yearly 약점이 가장 큰 개선 기회
- **블렌딩이 선택보다 확실히 우월** — Yearly에서 DOT+CES+4Theta 블렌드로 28.1% 캡처
- **DOT+CES 조합이 거의 모든 빈도에서 최적** — 4Theta/ETS 가중치는 대부분 0
- **빈도별 최적 비율이 다름** — Yearly DOT:CES=4:4, Monthly 8:2, Hourly 2:8
- **4Theta/ETS 폭발** — Weekly에서 4Theta OWA 2.05, ETS 77억(!) → 안전장치 필수
- **Daily는 프로파일링 가치 거의 없음** — 어떤 전략도 0.6%만 캡처
- **실용적 결론**: "선택"보다 "블렌딩", 블렌딩 비율을 빈도별로 설정하는 것이 현실적 전략

### E006
- **E005 비율은 과적합** — Yearly에서 CES 40%가 대규모에서 CES 0%로 반전 (+3.6% 악화)
- **재최적화 AVG OWA 0.876 (기존 0.886 대비 -1.1%)** — 소폭 개선은 확인
- **안정적으로 작동하는 빈도**: Monthly(DOT0.8+CES0.2), Hourly(DOT0.2+CES0.8)
- **Hourly가 유일한 유의미 개선** — 1.178→1.131 (-4.1%), CES 80% 비율이 대규모에서도 안정
- **Yearly는 DOT+4Theta** — CES Yearly OWA 0.984로 나쁨. DOT0.8+4Theta0.2=0.807
- **Weekly는 DOT 단독 최적** — 블렌드 불필요
- **소규모 최적 비율을 그대로 쓰면 위험** — 반드시 대규모 검증 필요

### E007
- **2-way 재확인**: DOT0.2+CES0.8 = OWA 1.131 (-4.1%), E006과 완전 일치
- **3-way 발견**: CES0.7+DTSF0.3 = OWA 1.082 (**-8.2%**!) DOT 가중치 0
- **DTSF 앙상블 가치**: 단독 OWA 1.307이지만 잔차 직교성(0.3)으로 블렌드에서 핵심 기여
- **안정성 문제**: 승률 47.1%, 중앙값 +0.006 → 소수의 큰 개선이 평균을 끌어내림
- **엔진 반영**: ensemble=True 시에만 CES+DTSF 블렌드 적용이 안전

### E008
- **Daily 블렌딩 완전 불가** — DOT 단독이 최적. CES/DTSF 어떤 비율도 개선 없음
- **DOT-CES 잔차 상관 0.987** — 거의 동일하게 틀림. Hourly(~0.9)와 근본적으로 다름
- **DTSF Daily 파괴**: OWA 1.784, 65.2% 시리즈에서 Naive2보다 나쁨. period=7에서 부적합
- **실패율 23.6% 구조적** — 시리즈 길이와 약한 상관만. DNA로 분리 불가
- **결론**: Daily 개선은 모델 자체 개선(다중 계절성, DL) 필요. 블렌딩/프로파일링으로 불가능

### E009
- **전처리 oracle gap -17.4%** — 모델 선택(-15.6%)보다 큰 이론적 가치
- **하지만 선택이 불가능** — DNA 특성↔전처리 개선의 상관 r<0.25
- **diff/stl은 high risk high reward** — 25.6%/12.8% 시리즈에서 최적이지만 나머지에서 폭발
- **규칙 기반 선택 OWA 1.091 (+34%)** — raw보다 대폭 악화! "잘못 고르면 폭발" 재확인
- **안전한 전처리는 raw뿐** — 현재 엔진의 "전처리 없음" 정책이 실제로 최적
- **결론**: 전처리 선택도 holdout 기반 검증이 필요. DNA 수준 정적 규칙으로는 불가능

### E010
- **Shape 24d < DNA 4d** — Ridge 정확도 24.4% vs 26.2%. 차원 증가가 정보 증가가 아님
- **클러스터 purity 비슷** — Shape k=12 0.320 vs DNA k=8 0.298. 유의미한 차이 없음
- **Cluster-best 전부 악화** — k=12에서도 OWA 0.889(+9.5%). 클러스터 내 전략 다양성이 너무 큼
- **Ridge(Combined) OWA 0.968(+19%)** — DNA+Shape 합쳐도 악화
- **핵심 교훈**: 문제는 표현(representation)이 아니라 매핑의 비결정성
  → 같은 형태라도 내부 구조(노이즈, 이상치)에 따라 최적 전처리가 달라짐
  → 정적 분류(사전 관찰 기반)는 구조적 한계. holdout 기반 동적 선택만이 유일한 경로

### E011
- **유의미 붕괴점**: median×2 threshold 적용 시 시리즈의 10% (평균 21.9개)
- **Clustering**: 붕괴점 간격 CV 1.314 > 랜덤 0.855. 격동기/고요기가 존재
- **방향 교대 73.8%**: 랜덤(50%)보다 높지만 필연(100%)은 아님
- **크기 자기상관 ACF(1)=0.187**: 큰 붕괴 뒤 큰 붕괴 (변동성 클러스터링)
- **최대 크기↔오차 r=-0.288**: 큰 변동 시리즈가 오히려 예측 쉬움 (역직관)

### E012
- **Force Balance 보정 전면 실패**: 15개 파라미터 조합 모두 악화
- **최선 decay=0.3**: OWA 0.9055 (-11.5%), 승률 15.8%
- **격동기에서 더 악화**: 교란이 큰 곳에 큰 보정 = 재앙 (OWA 1.07~3.05)
- **핵심 교훈 3가지**:
  (1) 73.8% 교대는 "다음 붕괴점"의 방향이지 "다음 시점"이 아님. 90% 시점에서 보정이 틀림
  (2) amplitude 추정이 불확실. ACF 0.187로는 크기 예측 불가
  (3) 타이밍(언제 다음 붕괴가 오는지)을 모르면 보정이 불가능
- **이론은 유지, 적용 방식 변경 필요**: 직접 보정이 아닌 다른 경로

### E013
- **Holdout 승자 OWA 1.006 (-24%)**: "시도→평가→선택"조차 실패
- **Holdout↔Test 상관 r=0.267**: holdout 1 period로는 미래 성능 예측 불가
- **Oracle OWA 0.602 (+25.8%)**: 이론적 여지는 거대하지만 접근 불가
- **Safe 전략 0.734**: holdout이 맞을 때는 진짜 좋다. 문제는 "언제 맞는지" 모른다
- **Holdout=Oracle 일치율 18.6%**: 9가지 중 랜덤(11%)보다 약간 높을 뿐
- **diff 전처리의 함정**: holdout에서 최다 선택되지만 test에서 폭발 → 과적합의 전형
- **핵심 교훈**: holdout 1 period는 미래의 대리변수로 불충분.
  시계열은 non-stationary이므로 가장 최근 데이터도 미래를 대표하지 않는다

### E014
- **잔차 ACF(1) = 0.768**: 95.4%가 |ACF|>0.2 → 잔차에 강한 자기상관 구조 존재
- **그러나 AR 보정은 전부 악화**: AR(1) -28.3%, AR(2) -28.4%, AR(3) -27.1%
- **승률 39~42%**: 동전 뒤집기보다 나쁨
- **ACF 높은 시리즈에서 더 악화**: 고ACF -29%, 저ACF -4~7% (역직관)
- **근본 원인**: seasonal naive 잔차(trainY[t]-trainY[t-12])의 ACF는 추세 잔류분
  DOT는 이미 추세+계절성을 캡처하므로 AR이 학습한 "구조"는 DOT와 무관
- **교훈**: 프록시 잔차와 실제 잔차의 괴리. DOT 잔차를 직접 추출하려면 refit이 필요하나 비용 과다

### E015
- **Rolling 2-fold safe = OWA 0.8008 (+0.96%)**: 15개 실험 중 **유일한 양의 개선**
- **1-fold → 2-fold**: 0.8136→0.8008 (rolling이 노이즈를 줄임)
- **2-fold → 3-fold**: 0.8008→0.8015 (추가 fold 이득 없음)
- **diff 포함 시 악화**: 3-fold all 0.8411(-4.02%). E013과 동일, diff 과적합 재확인
- **상관은 여전히 낮음**: r=0.285→0.299 (미미한 개선)
- **Oracle gap 캡처율 6.1%**: 15.66% gap 중 극소만 활용
- **핵심 교훈**: rolling holdout은 1-period보다 확실히 낫지만, +0.96%는 복잡도 대비 미미

## dataProfiling 연구 최종 결론 (E001~E015)

### 확실히 안 되는 것
- DNA → 모델 선택 (Ridge, 규칙 모두 실패)
- DNA → 전처리 선택 (규칙 기반 +34% 악화)
- Diff Shape → 전처리 선택 (DNA보다 나쁨, 24.4% < 26.2%)
- 소규모 최적 비율의 직접 적용 (과적합)
- Daily 빈도의 블렌딩/프로파일링 개선
- **정적 분류 전체** — 어떤 사전 관찰 기반 표현(DNA, Shape, Combined)도 전략 매핑 불가
- **붕괴점 기반 직접 보정** — 통계적 특성은 사실이나 예측 보정으로 변환 불가
- **Holdout 1-period 동적 선택** — 상관 r=0.267, 승자가 DOT보다 악화, diff 과적합
- **Seasonal naive 잔차 AR 보정** — 잔차에 구조가 있지만 DOT 잔차와 무관. 모든 차수 -27~28%
- **diff 전처리** — holdout이든 rolling이든 체계적 과적합. 어떤 검증 전략으로도 안전하게 쓸 수 없음

### 확실히 맞는 것
- 블렌딩 > 선택 (잘못 고르면 악화, 섞으면 안전)
- Hourly CES+DTSF 블렌드 = -8.2% (잔차 직교성)
- 전처리 oracle gap 25.8%/17.4% (이론적 가치 거대하지만 접근 불가)
- DOT-CES 잔차 상관은 빈도별로 다름 (Hourly ~0.9, Daily 0.987)
- **매핑의 비결정성** — 같은 형태/DNA라도 최적 전략이 다름
- **붕괴점에 구조 존재** — clustering, 73.8% 교대, 크기 자기상관
- **단일 시계열의 미래는 그 시계열의 과거로 결정되지 않는다**
- **Rolling holdout > single holdout** — 2-fold safe가 DOT 대비 +0.96% (유일한 양의 개선)

### 15개 실험의 메타 교훈
1. **"관찰 → 판단"은 안 된다** — DNA, Shape, 붕괴점, 잔차 ACF 모두 실패 (E001~E012, E014)
2. **"시도 → 선택"은 거의 안 된다** — holdout 1-period 실패(E013), rolling 2-fold만 +0.96%(E015)
3. **"섞기"만 안정적이다** — 블렌딩이 유일한 신뢰할 수 있는 개선 (E005~E007)
4. **Oracle gap은 대부분 접근 불가** — 25.8%의 이론적 여지 중 최대 6.1%만 캡처
5. **DOT-Hybrid는 이미 놀라울 정도로 좋다** — 15가지 시도 중 이긴 것은 rolling 2-fold의 +0.96%뿐
6. **프록시와 실제의 괴리** — seasonal naive 잔차 ≠ DOT 잔차, holdout 성능 ≠ test 성능
