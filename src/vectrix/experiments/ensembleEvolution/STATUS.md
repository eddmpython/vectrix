# ensembleEvolution — 차세대 앙상블 연구

## 연구 목표
잔차 직교성 기반 앙상블로 "나쁜 모델이 평균을 오염시키는" 문제 해결. E014에서 가중 앙상블이 단독 모델보다 나쁨을 확인 → 근본적으로 다른 앙상블 전략 필요.

## 채택된 변경사항
(아직 없음)

## 기각된 아이디어

1. **DNA 가중 앙상블 (E014, metaSelection)**: 5개 모델 가중 결합이 top-1 단독(36.78%)보다 나쁨(59~78%). 극단적 MAPE 모델이 평균을 오염시킴.
2. **잔차 직교 선택적 앙상블 (E018)**: 모든 모델 쌍의 잔차 상관이 0.73~1.0.
   |r|<0.3인 직교 쌍이 62개 중 6건(9.8%)만 발견. 직교 앙상블 승률 0/61.
   근본 원인: vectrix 통계 모델들이 유사한 예측 패턴 → 잔차가 동질적.
3. **Residual Boosting Forecaster (E029)**: Stage1→잔차→Stage2→잔차→Stage3 그래디언트 부스팅.
   1/10 승률. 근본 원인: vectrix 모델에 fittedValues() 인터페이스 없음 → predict(n)은 미래 예측이라 in-sample 잔차 추출 불가.
4. **Regime-Switching Forecaster (E030)**: CUSUM 변동점 감지 → 레짐별 모델 선택.
   2/12 승률 16.7%. regimeShift 데이터에서도 dot(5.85%)에 밀림(5.88%).
   multi_regime(validation 기반)이 regime_switch(규칙 기반)보다 나음 (3.17 vs 3.50).
   근본 원인: 마지막 레짐만 사용 → 학습 데이터 감소 → 정보 손실.

## 핵심 인사이트

- **모델 간 잔차 다양성이 근본적으로 부족**: 통계 모델만으로는 앙상블 효과 불가능
- garch/auto_ces/croston/naive/window_avg: 잔차 상관 1.0 (사실상 동일 예측)
- mstl만 유일하게 상대적으로 낮은 상관 (0.73~0.78)
- 앙상블이 도움되려면 "다르게 틀리는" 모델 필요 → ML 모델이 유일한 답
- **통계 모델 앙상블 연구는 가성비 낮음**, 모델 선택(metaSelection)에 집중이 더 효과적
- **Residual Boosting 전제 붕괴**: fittedValues() 없이 부스팅 불가. 인터페이스 추가 후 재실험 필요
- **레짐 전환 모델의 정보 손실**: 데이터 절단(마지막 레짐만 사용)은 학습 데이터 감소로 역효과. "레짐별 가중치" 방식이 대안
- **multi_regime > regime_switch**: validation 기반 모델 선택이 규칙 기반보다 우월
