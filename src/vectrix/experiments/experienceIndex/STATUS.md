# experienceIndex — 경험 역인덱스 기반 예측 전략 학습

## 정체성

파운데이션 모델은 "예측 자체"를 학습한다. 우리는 "예측 전략"을 학습한다.
대량의 시리즈에 대해 실제로 예측을 수행하고, 그 경험을 역인덱스로 축적한다.
새 시리즈가 들어오면 비슷한 경험을 검색하여 전략을 가져온다.

## 핵심 아이디어

### 검색 엔진의 역인덱스에서 착안

정방향: 시리즈 → DNA 특성 (분석)
역방향: DNA → 경험 목록 (조회)

### dataProfiling과의 차이

- dataProfiling: DNA → 모델/전처리를 "매핑 함수"로 학습 (Ridge, 규칙). 실패함
- experienceIndex: DNA를 키로 "경험 자체"를 저장하고 "검색"한다. 매핑 함수 없음

### 왜 작동하는가 (E002에서 확인)

1. **DNA 65차원이 결정적** — 4차원 간이 DNA는 시리즈를 구분 못 함. 65차원은 가능
2. **검색이 매핑보다 낫다** — Ridge(전역 함수)는 실패했지만 kNN(국소 검색)은 성공
3. **경험이 축적될수록 개선** — 500개 +3.1% → 4500개 +4.6%, 학습 곡선 미포화
4. **선택 > 블렌딩** — 경험 기반 "모델 선택"은 작동, "가중 블렌딩"은 역효과

## 경험 DB 저장 구조 (설계)

```
data/experience/
├── schema.json          ← DNA 특성 목록, 모델 목록, 버전
├── monthly.jsonl        ← Monthly 경험 로그 (append-only, git 추적)
├── yearly.jsonl         ← Yearly
├── quarterly.jsonl
├── weekly.jsonl
├── daily.jsonl
├── hourly.jsonl
└── _build/
    └── monthly.parquet  ← 빌드된 검색용 파일 (.gitignore)
```

- `.jsonl`은 git에 올린다 — append-only, diff 가능
- `.parquet`은 로컬 빌드 — 검색 속도용, `.gitignore`
- `schema.json`으로 스키마 변경 추적
- 빈도별 분리 — 같은 빈도 안에서만 매칭

## 핵심 발견

### E001
- 간이 DNA 4차원(seasonality, trend, acf1, lengthRatio)으로는 시리즈 구분 불가
- 버킷 purity 최대 0.348 (8후보 중 35%만 일치)
- kNN 전부 악화: k=5 -2.84%, k=50 -0.31%
- 버킷 역인덱스: ±0.00% (DOT와 동일)
- **교훈**: DNA 차원이 부족하면 검색도 안 된다

### E002 ★★★
- **65d DNA + 5000개 경험으로 kNN +3.72% 달성!**
- dataProfiling 15개 실험 최고(rolling holdout +0.96%)의 **4배**
- kNN(k=50) OWA 0.8279, Oracle gap 17.1% 캡처
- 학습 곡선 단조 증가: 500개 +3.1% → 4500개 +4.6% (미포화)
- 경험 "선택"은 성공(+4.6%), 경험 "블렌딩"은 실패(-10.6%)
- **E001→E002의 반전**: 4d에서 전부 악화 → 65d에서 전부 개선. DNA 차원이 핵심

### E003
- **20K 규모에서 학습 곡선 미포화 확인** — 15K DB +3.98% (캡처 18.1%)
- 도메인 순도 40.6% (랜덤 22.8%의 1.78x) — DNA가 자연 분리하지만 불완전
- **도메인 분리 불필요** — 도메인 내 경험이 전체보다 -0.65% 나쁨 (DB 크기 > 도메인 순도)
- 도메인별 성능 차이 큼: Industry +8.0%, Macro +5.7%, Finance +3.8%, Demographic +1.2%
- DOT이 약한 도메인에서 경험 선택 효과 극대화
- **결론**: DB 크기를 키우는 것이 도메인 분리보다 중요. 도메인 태그 불필요

### E004
- **모든 빈도에서 경험 인덱스 유효!** Yearly +7.18%, Quarterly +2.82%, Monthly +3.98%
- DOT 기준 OWA가 높을수록(나쁠수록) 개선이 큼 — Yearly(0.93) > Monthly(0.86) > Quarterly(0.84)
- Yearly가 가장 큰 개선 — 계절성 없는 데이터에서 모델 선택이 더 중요
- k값은 빈도별로 다를 수 있음: Quarterly k=20이 k=50보다 좋음
- 학습 곡선은 모든 빈도에서 단조 증가 (대체로)

### E005
- 후보 8→20개: Oracle gap 22.1%→32.6% (+10.5%p) 확대
- kNN 선택: 8후보 +4.38%, 20후보 +4.57% — 순효과 +0.19%p로 미미
- Oracle은 커졌지만 kNN이 그 확대분을 캡처하지 못함
- ESN이 Oracle 최적 14.3% (DOT 다음 2등) — 높은 다양성 기여
- **결론**: 8후보로 충분. 계산 비용 2.5배 대비 이득 0.2%p는 비효율적

### E006 ★★
- **Top 50 특성이 전체 65개보다 +1.13% 좋다!** (OWA 0.8228 vs 0.8323)
- 하위 15개 특성이 kNN 검색에 노이즈로 작용
- trend(0.00)와 basic statistics(0.00) 그룹이 완전히 무용
- forecastability(0.48), seasonality(0.41)이 가장 중요한 그룹
- 개별 1위: seasonal_peak_position (1.241) — 계절 피크 위치가 모델 선택의 핵심 단서
- **결론**: DNA 특성 선별로 추가 개선 가능. 50d > 65d (단, E007에서 대규모/타빈도 검증 시 불안정 확인)

### E007
- **Top 50은 범용적이지 않다** — 5K Monthly에서만 +0.53%, 대규모/타빈도에서 불안정
- 20K Monthly 전체: 65d 0.8194 vs 50d 0.8217 → 65d가 -0.28% 나음
- 20K Monthly 학습 곡선: 15K DB에서만 50d가 +1.88% 우위 (다른 크기에서는 혼재)
- Yearly: 거의 동등 (-0.07%)
- Quarterly: 65d가 일관되게 우위 (-0.41~-0.58%)
- **Permutation importance 불안정**: E006 Top 5 (seasonal_peak_position 등)가 이번에는 제거 대상에 포함
- **실무 결론**: 65d를 그대로 사용하는 것이 안전. 차원 축소는 복잡도 대비 이득 불확실
