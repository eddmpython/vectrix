# foundationAttack — 파운데이션 모델 격파 전략

## 정체성

이 실험 폴더는 **파운데이션 모델(Chronos-2, TimesFM, Moirai 등)을 통계 기반 지능으로 이기는 연구**다.

dataProfiling이 "데이터가 무엇인가"를 물었다면,
이 폴더는 그 다음 질문을 던진다: **"데이터를 이해한 뒤, 어떻게 예측을 조립하는가?"**

## 핵심 전략 — Learned Orchestration

파운데이션 모델은 수십억 데이터포인트로 **예측 자체를 학습**한다.
우리는 그것을 따라하지 않는다.

대신 **오케스트레이션을 학습**한다:
- 어떤 데이터인지 파악하고 (Learned Profiling)
- 어떤 모델/비율이 최적인지 선택하고 (Learned Selection)
- 예측 급변 구간을 잘라서 전략을 바꾼다 (Learned Surgery)

```
파운데이션 모델:  데이터 → [거대 신경망] → 예측
우리의 전략:      데이터 → [프로파일링] → [선택/블렌딩] → [급변 절개] → 예측
                          학습된 특성     학습된 가중치    학습된 분기점
```

## 적의 현황 — 알아야 이긴다

|------|------|------|------|
| 1 | TimeCopilot | 앙상블 | Chronos-2 + TimesFM + TiRex 미디언 앙상블 ($30) |

### 파운데이션 모델의 약점 (우리의 기회)
1. **고빈도 데이터(초단위)에서 통계 모델이 승리** — GIFT-Eval에서 확인됨
2. **도메인 특화 부족** — 범용 학습이라 특정 패턴에 과소적합
3. **간헐적 수요(intermittent)에 취약** — 0이 많은 시리즈 처리 미숙
4. **급변(regime change) 대응 느림** — 사전학습된 분포에 갇힘
5. **컴퓨트 비용** — 추론에 GPU 필요, 우리는 CPU 밀리초

### M4 vs GIFT-Eval
| 차원 | M4 | GIFT-Eval |
|------|----|----|
| 시리즈 수 | 100K | 144K+ |
| 데이터포인트 | 1M | 177M |
| 빈도 | 6개 | 10개 (초~연) |
| 도메인 | 비공개 혼합 | 7개 명시 (에너지, 소매, 교통...) |
| 메트릭 | OWA (MASE+sMAPE) | MASE, WQL, CRPS |
| 의미 | 기존 모델 검증 | 파운데이션 모델 경쟁장 |

## 단계별 공략

### Phase 0: 무기 점검 (사전 준비)
**목적**: 현재 우리 무기(모델/프로파일링)의 정확한 전력을 파악

| 실험 | 내용 | 성공 기준 |
|------|------|----------|
| 001 | GIFT-Eval 데이터셋 확보 + 포맷 파악 | 7개 도메인 데이터 로드 가능 |
| 002 | 현재 DOT-Hybrid를 GIFT-Eval에서 돌려보기 | 도메인별 MASE 기준선 확보 |
| 003 | Chronos-2 / TimesFM GIFT-Eval 재현 | 적의 실제 성능 수치 확보 |

**데이터 출처**: HuggingFace `Salesforce/gift_eval` 데이터셋
**기록 필수**: 각 실험에서 사용한 데이터셋, 시리즈 수, 빈도, 도메인을 docstring에 명시

### Phase 1: Learned Profiling — DNA 특성 강화
**목적**: 데이터에서 학습하여 DNA 특성 벡터의 분별력을 높인다

| 실험 | 내용 | 성공 기준 |
|------|------|----------|
| 004 | GIFT-Eval 7개 도메인 DNA 프로파일 추출 + 클러스터 분석 | 도메인간 DNA 분포 차이 확인 |
| 005 | 도메인-DNA 매핑 학습 (DNA → 도메인 분류) | 분류 정확도 > 70% |
| 006 | DNA 특성 선별 — 모델 성능과 상관 높은 특성만 추출 | Top-10 특성으로 Oracle gap 20%+ 캡처 |
| 007 | DNA 특성 증강 — 기존 65개 + 새 특성 (wavelet, 엔트로피 등) | 분별력 개선 확인 |

**dataProfiling과의 차이**: dataProfiling은 M4 내부 관찰이었다면, 여기서는 **외부 도메인**(GIFT-Eval)에서의 DNA 일반화를 검증

### Phase 2: Learned Selection — DNA → 최적 전략 매핑
**목적**: DNA 프로파일로부터 최적 모델/블렌딩 비율을 자동 결정

| 실험 | 내용 | 성공 기준 |
|------|------|----------|
| 008 | 도메인별 Oracle 모델 분포 + Oracle gap 측정 | 어떤 도메인에서 기회가 큰지 파악 |
| 009 | DNA → 최적 모델 매핑 (비선형: XGBoost/LightGBM) | Ridge(36.7%)보다 +10%p 이상 |
| 010 | DNA → 블렌딩 비율 매핑 (회귀 문제) | 고정 비율 대비 MASE -3% 이상 |
| 011 | 도메인 특화 선택 전략 (도메인 인식 라우팅) | 범용 전략 대비 -5% MASE |

**metaSelection과의 차이**: metaSelection은 M4 Monthly만 대상이었고 Ridge 선형만 시도했다. 여기서는 비선형 + 다빈도 + 다도메인

### Phase 3: Learned Surgery — 예측 급변 절개
**목적**: 시리즈 내 레짐 변화를 감지하고, 구간별 다른 전략을 적용

| 실험 | 내용 | 성공 기준 |
|------|------|----------|
| 012 | GIFT-Eval 시리즈에서 레짐 변화점 탐지 (PELT, BOCPD) | 변화점이 있는 시리즈 비율 파악 |
| 013 | 변화점 전후 최적 모델이 다른지 검증 | 전/후 Oracle 모델 불일치 > 30% |
| 014 | 구간별 독립 예측 + 접합 파이프라인 | 단일 전략 대비 MASE -3% 이상 |
| 015 | DNA 기반 변화점 예측 (변화점이 올 것인가?) | 이진 분류 AUC > 0.7 |

**이전 실패와의 차이**: E030/E039/E040에서 변화점/레짐 모델이 실패했지만, 그때는 "레짐별 모델을 만들려" 했다. 여기서는 "레짐별로 기존 모델 선택을 바꾸는" 것

### Phase 4: Integration — 통합 파이프라인
**목적**: Phase 1~3의 성과를 하나의 파이프라인으로 결합

| 실험 | 내용 | 성공 기준 |
|------|------|----------|
| 016 | Phase 1~3 통합 파이프라인 (profile → select → surgery → predict) | 도메인별 파운데이션 모델 1개 이상 격파 |
| 017 | M4 역검증 — 통합 파이프라인이 M4에서도 개선되는지 | M4 AVG OWA < 0.848 유지 |
| 018 | 속도 벤치마크 — CPU 밀리초 vs GPU 파운데이션 추론 | 100x+ 속도 우위 확인 |

### Phase 5: Domain Kill — 도메인별 격파
**목적**: 특정 도메인에서 파운데이션 모델을 이긴 뒤 점진적 확장

우선순위 도메인 (기회 순):
1. **에너지** — 계절성 강하고 패턴 반복적, 통계 모델의 강점
2. **소매/수요** — 간헐적 수요 포함, Croston/CES 활용 가능
3. **교통** — 다중 계절성(일/주), AutoMSTL/TBATS 강점
4. **금융** — GARCH 계열의 변동성 모델링 강점

| 실험 | 내용 | 성공 기준 |
|------|------|----------|
| 019 | 에너지 도메인 특화 파이프라인 | Chronos-2 에너지 MASE보다 낮음 |
| 020 | 소매 도메인 특화 파이프라인 | 소매 도메인 MASE 상위 5위 이내 |
| 021 | 교통 도메인 특화 파이프라인 | 교통 도메인 격파 |
| 022 | 4개+ 도메인 격파 시 → GIFT-Eval 종합 순위 도전 | 종합 상위 10위 이내 |

## 데이터 리니지 (실험별 기록)

모든 실험에서 다음을 docstring에 명시적으로 기록:

```
데이터 리니지:
- 출처: GIFT-Eval / M4 / 내부 샘플
- 도메인: 에너지 / 소매 / 교통 / ...
- 빈도: H / D / W / M / Q / Y
- 시리즈 수: N개
- 기간: YYYY-MM ~ YYYY-MM
- 전처리: 원본 그대로 / 정규화 / 결측 보간
- 학습/검증 분할: train N% / valid N% (또는 expanding window)
- 시드: 42 (재현 가능)
```

## 성공 정의

### 최소 성공 (Minimum Viable)
- GIFT-Eval **1개 도메인**에서 Chronos-2보다 낮은 MASE
- M4 AVG OWA 0.848 이하 유지

### 목표 성공
- GIFT-Eval **3개+ 도메인**에서 파운데이션 모델 1개 이상 격파
- M4 AVG OWA < 0.840 (현재 0.848에서 -1%)

### 궁극 목표
- GIFT-Eval 종합 상위 10위
- "GPU 없이 CPU만으로 파운데이션 모델 격파" — 킬러 마케팅

## 원칙

1. **도메인별 각개격파** — 전체를 한 번에 이기려 하지 않는다. 쉬운 도메인부터.
2. **학습은 선택에 집중** — 예측 자체를 학습하지 않는다. "무엇을 쓸지"를 학습한다.
3. **기존 무기를 최대 활용** — 새 모델을 만들기보다, 22개 기존 모델의 조합을 극대화한다.
4. **데이터 기록 철저** — 모든 실험에서 무슨 데이터를 봤는지 반드시 기록한다.
5. **실패도 자산** — dataProfiling E001~E006, metaSelection E012~E016의 실패에서 배운다.
